# 游戏改进说明（v2.0）

## 新增功能

### 1. ✅ 限制小鸟数量
- 每局游戏限制使用 **5只小鸟**
- 实时显示剩余小鸟数量
- 用完小鸟后游戏结束并显示失败界面

**实现位置**：`src/entities/Slingshot.ts`
```typescript
private maxBirds: number = 5;
```

### 2. ✅ 降低小猪击倒难度
采用多重检测机制，让小猪更容易被击倒：

#### a) 降低倾斜角度阈值
- **原来**：倾斜超过45度
- **现在**：倾斜超过30度即被击倒

#### b) 新增速度检测
- 当小猪移动速度超过 **3m/s** 时，即判定为被击中
- 这样直接撞击也能击倒小猪，无需等待倾斜

#### c) 提高检测频率
- **原来**：每0.5秒检查一次
- **现在**：每0.3秒检查一次

**实现位置**：`src/entities/Pig.ts`
```typescript
// 角度检测
if (angleX > 30 || angleZ > 30) {
  this.die();
}

// 速度检测
const speed = Math.sqrt(velocity.x * velocity.x + velocity.y * velocity.y + velocity.z * velocity.z);
if (speed > 3) {
  this.die();
}
```

### 3. ✅ 建筑破坏系统
为建筑方块添加了完整的破坏系统：

#### 生命值系统
- 每个方块有 **100点生命值**
- 根据碰撞速度计算伤害
- 生命值归零后方块被破坏

#### 视觉反馈
- **健康状态**：棕色
- **受损状态**：颜色逐渐变暗
- **破碎状态**：深灰色半透明

#### 破坏效果
- 施加随机力产生爆炸效果
- 2秒后自动销毁实体

**实现位置**：`src/entities/Block.ts`
```typescript
private checkDamage() {
  const speed = calculateSpeed(velocity);
  if (speed > 2) {
    const damage = (speed - 2) * 15;
    this.takeDamage(damage);
  }
}
```

### 4. ✅ 小猪死亡后销毁
- 小猪被击倒后变灰并施加向上的力（弹起效果）
- **1.5秒后**自动销毁实体
- 避免场景中累积过多死亡物体

**实现位置**：`src/entities/Pig.ts`
```typescript
private die() {
  this.isDead = true;
  // 施加向上的力
  this.collider.applyForce(new Vector3(0, 5, 0));
  // 1.5秒后销毁
}
```

### 5. ✅ 对象池管理系统
创建了通用的对象池管理系统：

#### 功能特性
- **对象复用**：减少频繁创建销毁的开销
- **预创建**：支持预分配对象
- **生命周期管理**：自动激活/禁用对象
- **统计信息**：跟踪活动和可用对象数量

#### 两种实现
1. **通用对象池**：`ObjectPool<T>`
   - 支持任意类型
   - 可自定义创建、重置和生命周期回调

2. **实体对象池**：`EntityPool`
   - 专门用于Entity管理
   - 自动处理isActive状态

**实现位置**：`src/managers/ObjectPool.ts`

**使用示例**：
```typescript
// 创建对象池
const pool = new ObjectPool<MyObject>(
  () => new MyObject(),           // 创建函数
  (obj) => obj.reset(),            // 重置函数
  10                               // 预创建10个
);

// 获取对象
const obj = pool.get();

// 归还对象
pool.release(obj);
```

## 游戏平衡调整

### 物理参数优化

| 对象 | 参数 | 原值 | 新值 | 说明 |
|------|------|------|------|------|
| 小猪 | mass | 2.0 | 1.0 | 更轻，更容易推倒 |
| 建筑 | mass | 5.0 | 1.0 | 更容易破坏 |
| 小猪 | 倾斜阈值 | 45° | 30° | 更容易判定倒下 |
| 小猪 | 速度检测 | 无 | 3m/s | 新增撞击检测 |

### 游戏节奏优化
- **小鸟数量限制**：增加策略性，需要精准瞄准
- **快速反馈**：降低检测间隔，即时响应玩家操作
- **视觉反馈**：建筑受损变色，小猪死亡弹起

## 技术细节

### 1. 碰撞伤害计算
```typescript
// 速度转换为伤害
const damage = (speed - threshold) * multiplier;

// threshold: 2m/s (低于此速度不造成伤害)
// multiplier: 15 (伤害系数)
```

### 2. 颜色渐变算法
```typescript
// 根据生命值百分比插值颜色
const healthPercent = health / maxHealth;
const r = startColor.r * healthPercent + endColor.r * (1 - healthPercent);
```

### 3. 对象销毁延迟
```typescript
// 避免立即销毁造成的闪烁
if (this.isDead) {
  this.deadTimer += deltaTime;
  if (this.deadTimer > 1.5) {
    this.entity.destroy();
  }
}
```

## 已知限制

1. **对象池未全面应用**：当前仅创建了对象池系统，但小鸟、小猪和建筑还未使用对象池（为简化实现）
2. **PhysX依赖**：需要完整的物理引擎支持，不支持LitePhysics
3. **移动端性能**：极多物体时可能有性能问题

## 未来改进方向

1. **完全应用对象池**：将所有游戏对象纳入对象池管理
2. **粒子效果**：添加击中特效、爆炸效果
3. **音效系统**：碰撞音、发射音、胜利音
4. **关卡系统**：多个关卡，难度递增
5. **物理材质**：不同材质的建筑（木头、石头、玻璃）
6. **小鸟技能**：多种小鸟类型（爆炸、分裂、加速）
